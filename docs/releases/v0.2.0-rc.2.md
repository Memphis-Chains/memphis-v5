# v0.2.0-rc.2 — Production Ops Baseline

Date: 2026-03-09
Tag: `v0.2.0-rc.2`

## Scope
Operational hardening for local Ollama production path.

## Included
- OLLAMA bridge as user systemd service
- Bridge healthcheck timer + auto-restart recovery
- One-command runtime smoke (`npm run smoke:ollama-runtime`)
- Nightly self-hosted runtime smoke workflow
- Failure alerting with dedup + optional webhook
- Recovery auto-close for incident issue
- Weekly KPI + weekly recovery drill workflows
- 5-minute operator runbook
- Baseline freeze policy after rc.2

## Known limits
- Cloud/shared provider path not configured in this profile
- Runtime depends on local Ollama daemon availability
- Gateway `/ops/status` remains topology-dependent

## Rollback hint
- Keep `DEFAULT_PROVIDER=local-fallback` as immediate fallback if bridge path degrades.
- Restart bridge service: `systemctl --user restart ollama-compat-bridge.service`
