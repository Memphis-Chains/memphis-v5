# memphis-v4 v0.2.0-rc.3 — User-friendly install and release readiness

## Why this release
This release improves GitHub onboarding so a new user can clone, install, run, and validate Memphis quickly.

## Highlights
- Added user-facing quickstart guide:
  - `docs/USER-QUICKSTART-GITHUB.md`
- Improved README landing experience with explicit 5-minute start path.
- Clarified first-run commands and operator entry points for CLI/TUI.

## Install (recommended)
```bash
git clone https://github.com/Memphis-Chains/memphis-v4.git
cd memphis-v4
./scripts/install.sh
npm run build
npm test
```

## First run
```bash
npx tsx src/infra/cli/index.ts doctor --json
npx tsx src/infra/cli/index.ts ask --input "Hello Memphis"
```

## Product status
- Core product flow is ready for users (install, run, ask, operate).
- External two-host production proof remains infrastructure-dependent (requires second real host).
- Local-ready external-proof workflow is available and hardened.
