# PROJECT CHARTER — Memphis v4

Data: 2026-03-08  
Status: Draft v1 (na bazie ustaleń z Elathoxu)

## 1) Cel projektu (biznesowo-produktywny)
Dowieźć działający, stabilny i użyteczny produkt Memphis v4, który realnie wspiera codzienny biznes użytkownika.  
Motto: **„Moje życie to mój biznes.”**

## 2) Zakres MVP (scope)
Zakres MVP opieramy o **cały blueprint** z katalogu secret intake jako wzorzec implementacji.

Założenia wykonawcze:
- blueprint jest głównym źródłem wymagań,
- puste pola w blueprintcie będą uzupełniane wspólnie,
- uzupełnienia robimy zgodnie z best practices,
- priorytet: **jakość > szybkość**.

## 3) Out-of-scope / ograniczenia etapu
- Nie wchodzimy teraz w „straszne”/nadmiernie złożone rozwiązania.
- Nie over-engineerujemy.
- Celem etapu jest **wersja podstawowa, ale działająca** (working product).
- Rozbudowa i zaawansowane funkcje dopiero po domknięciu podstawy.

## 4) Definition of Done (Etap 1)
Etap uznajemy za domknięty, gdy:
1. Produkt działa end-to-end w wersji podstawowej.
2. Jest zgodny z blueprintem (w granicach uzgodnionych uzupełnień).
3. Spełnia uzgodniony baseline jakości.
4. **Elathoxu formalnie akceptuje gotowy działający produkt.**

---

## Zasady operacyjne dla tej sesji
- Nie przechodzimy do kolejnego etapu bez pełnego domknięcia bieżącego.
- Wszystkie materiały wejściowe zapisujemy w: `.memphis-intake/blueprint-secret/`.
- Każda decyzja projektowa powinna być uzasadniona pod kątem: stabilność, prostota, utrzymanie.
